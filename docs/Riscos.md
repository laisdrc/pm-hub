# Matriz de Riscos

## Objetivo
Identificar possíveis riscos que possam impactar no desenvolvimento do meu projeto PM Hub e definir ações para reduzir os efeitos.

| ID | Risco | Probabilidade | Resposta |
|----|--------|---------------|----------|
| R01 | Falta de tempo para desenvolvimento devido à faculdade e trabalho | Alta | Priorizar funcionalidades essenciais da versão 1.0 |
| R02 | Dificuldades na integração com Supabase | Média | Realizar testes de integração desde o início do desenvolvimento |
| R03 | Problemas na integração com Power BI | Média | Implementar o dashboard somente após os módulos principais estarem concluídos |
| R04 | Necessidade de aprender novas tecnologias durante o projeto | Alta | Reservar tempo para estudo e consulta à documentação oficial |
| R05 | Erros identificados durante os testes | Média | Utilizar Cypress para validação contínua das funcionalidades |
| R06 | Aumento excessivo do escopo durante o desenvolvimento | Alta | Definir claramente os requisitos da versão 1.0 e registrar melhorias para versões futuras |


## Riscos Prioritários
Os riscos com maior prioridade para acompanhamento são:

* R01 – Falta de tempo disponível
* R02 – Dificuldades na integração com Supabase
* R05 – Erros identificados durante os testes

## Revisão dos Riscos
A matriz de riscos deverá ser revisada sempre que houver mudanças significativas no escopo ou cronograma do projeto.
